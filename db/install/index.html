    Installation Details

    Database

    1. Add the PostgreSQL repository to your Ubuntu server's sources list:
    sudo sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'
wget -qO- https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add -
    1. Update the package lists and install the latest version of PostgreSQL:
    sudo apt update
sudo apt install -y postgresql postgresql-contrib
    1. Check the status of the PostgreSQL service:
    sudo systemctl status postgresql
    1. If the service is not running, start it:
    sudo systemctl start postgresql
    1. (Optional) Enable the firewall and open the PostgreSQL port for incoming connections:
    sudo ufw enable
sudo ufw allow 5432/tcp
sudo ufw allow 5432/udp
sudo ufw status
    1. Connect to the PostgreSQL server using the psql command-line tool:
    sudo -u postgres psql

    This will open the PostgreSQL interactive terminal, where you can execute SQL commands and manage your databases.

    That's it! You should now have a working PostgreSQL server installed on your Ubuntu machine.

    Extensions

    pgroonga

    Starting page on the installation in general. For installing on Ubuntu-based service, follow How to install for the official PostgreSQL section on this page. Copy of the instruction:

    $ sudo apt install -y software-properties-common
$ sudo add-apt-repository -y universe
$ sudo add-apt-repository -y ppa:groonga/ppa
$ sudo apt install -y wget lsb-release
$ wget https://packages.groonga.org/ubuntu/groonga-apt-source-latest-$(lsb_release --codename --short).deb
$ sudo apt install -y -V ./groonga-apt-source-latest-$(lsb_release --codename --short).deb
$ echo "deb http://apt.postgresql.org/pub/repos/apt/ $(lsb_release --codename --short)-pgdg main" | sudo tee /etc/apt/sources.list.d/pgdg.list
$ wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add -
$ sudo apt update
$ sudo apt install -y -V postgresql-15-pgdg-pgroonga # update to actual version of Postgres

    CREATE EXTENSION pgroonga;

    pg_cron

    Official page.

    sudo apt-get -y install postgresql-16-cron # update to actual version of Postgres

    Update DB configuration (postgresql.conf):

    shared_preload_libraries = 'pg_cron'
cron.database_name = 'postgres' # set to actual name of the db
cron.timezone = 'PRC' # optionally set the TZ

    CREATE EXTENSION pg_cron

    pg_prewarm

    1. Connect to your PostgreSQL database using the psql command-line tool:
    psql -U your_username -d your_database

    Replace your_username with your PostgreSQL username and your_database with the name of the database where you want to install the extension.

    1. Once connected to the database, execute the following SQL command to install the pg_prewarm extension:
    CREATE EXTENSION pg_prewarm;
    1. Verify that the extension is installed by running:
    SELECT * FROM pg_extension WHERE extname = 'pg_prewarm';

    This should return a row with information about the pg_prewarm extension version and schema.

    1. To use the pg_prewarm extension, you can call the pg_prewarm function to load relation data into the operating system cache or the PostgreSQL buffer cache. For example:
    -- Load a specific table into the OS cache
SELECT pg_prewarm('my_table');

-- Load an index into the OS cache
SELECT pg_prewarm('my_index');

-- Load a table and its indexes into the PostgreSQL buffer cache
SELECT pg_prewarm('my_table', 'buffer');

    The pg_prewarm function accepts the name of a table or index as the first argument and an optional second argument to specify the cache type ('os' for OS cache, 'buffer' for PostgreSQL buffer cache).

    1. You can also use the autoprewarm configuration parameter to automatically prewarm relations after a server restart. Add the following line to your postgresql.conf file:
    autoprewarm = on

    This will enable automatic prewarming of relations that were previously loaded into the OS cache.

    vector & cube

    1. Connect to your PostgreSQL database using the psql command-line tool:
    psql -U your_username -d your_database

    Replace your_username with your PostgreSQL username and your_database with the name of the database where you want to install the extensions.

    1. Once connected to the database, execute the following SQL commands to install the vector and cube extensions:
    CREATE EXTENSION vector;
CREATE EXTENSION cube;
    1. Verify that the extensions are installed by running:
    SELECT * FROM pg_extension WHERE extname IN ('vector', 'cube');

    This should return two rows, one for each extension, with information about their versions and schema.

    1. You can now use the vector and cube data types and functions in your database. For example:
    CREATE TABLE points (
    id SERIAL PRIMARY KEY,
    coords cube
);

INSERT INTO points (coords) VALUES ('(1,2,3)'), ('(4,5,6)');

SELECT * FROM points WHERE coords <-> '(0,0,0)'::cube < 5;

    This creates a table named points with a cube column, inserts two rows with 3D coordinates, and selects the points that are within a distance of 5 units from the origin.

    That's it! You have now installed and can use the vector and cube extensions in your PostgreSQL database.
