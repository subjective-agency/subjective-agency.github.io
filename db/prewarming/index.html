    Relations Data Prewarming

    Prewarming refers to the process of loading certain relation into either OS cache or database buffer. The best practice is to perform prewarming on server start up. To this end, following setup was implemented.

    Starter script

    bash script was created at /var/soft/prewarm_db.sh with following content (and make sure it's executable):

    #!/bin/bash

DB_NAME="wapadb"
DB_USER="warp"

prewarm_relations() {
    echo "Prewarming database relations..."
    psql -U $DB_USER -d $DB_NAME <<EOF
    -- THIS IS THE PLACE TO PUT PREWARMING STATEMENTS -- SEE BELOW
EOF
}

# Function to start the database
start_database() {
    echo "Starting PostgreSQL database..."
    sudo systemctl start postgresql
}

# Function to restart the database
restart_database() {
    echo "Restarting PostgreSQL database..."
    sudo systemctl restart postgresql
}

# Check the command line argument
case "$1" in
    start)
        start_database
        prewarm_relations
        ;;
    restart)
        restart_database
        prewarm_relations
        ;;
    *)
        echo "Usage: $0 {start|restart}"
        exit 1
        ;;
esac

echo "Database operation completed."

    Update bash Config

    Following function was added to the ~/.bashrc:

    function systemctl() {
    if [[ "$1" == "restart" && "$2" == "postgresql" ]]; then
        /var/soft/prewarm_db.sh restart
    else
        command systemctl "$@"
    fi
}

    With this, running systemctl restart postgresql or systemctl start postgresql would be intercepted, and the prewarming script would be run instead.
